The data is heavily imbalanced by non-downloaders activity. This will require rebalancing with weights. Here is the model used for fraud detection which is applicable to this Click-Through Rate prediction.

$ Fraud\_ Ratio = \frac{N_{fraud}}{N_{legit} + N_{fraud}} $
$ Weights = \frac{1}{Fraud\_Ratio}$

Conclusion

Overall, each feature have has a good level of distinction with variability. The only exception is the Hour's feature where the distinction is the least unique. With this, we can simply remove the Hour's variable without incurring much loss of information when training our model.
